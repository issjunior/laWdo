# Instruções para Claude Code

## Idioma do Projeto

**OBRIGATÓRIO**: Todo o desenvolvimento deste projeto deve ser realizado em **português do Brasil**.

### O que deve estar em português do Brasil:

1. **Diálogos com o usuário** - Todas as interações, perguntas, respostas
2. **Arquivos gerados** - Código-fonte, documentação, configurações
3. **Comentários no código** - Comentários em arquivos TypeScript, JavaScript, CSS, etc.
4. **Documentação** - README, guias, manuais, especificações
5. **Interface do usuário** - Textos, mensagens, labels, botões
6. **Mensagens de erro/log** - Mensagens visíveis para usuários
7. **Nomes de variáveis/funções** - Quando representam conceitos do domínio (ex: `usuario`, `laudo`, `perito`, `solicitante`)

### Contexto:

Este projeto é o **laWdo** desenvolvido para a **Polícia Científica do Paraná (PCP)**. É utilizado por **peritos forenses** para criação e gerenciamento de laudos periciais. Todo o domínio do projeto, terminologia técnica e contexto organizacional são em português do Brasil.

### Exceções permitidas:

- **Nomes de bibliotecas/frameworks externos**: Mantenha em inglês (React, TypeScript, Electron, etc.)
- **Termos técnicos padrão da indústria**: Podem permanecer em inglês quando não houver equivalente comum em português
- **Código de terceiros**: Não modifique apenas para tradução
- **Comandos técnicos/scripts**: npm, git, bash commands permanecem em inglês

### Exemplos corretos:

```typescript
// CORRETO - em português do Brasil
const usuario = await buscarPerito(id); // Comentário em português
const laudo = criarLaudoPericial(dados); // Outro comentário

// ERRADO - em inglês
const user = await findExpert(id); // Comment in English
const report = createForensicReport(data); // Another comment
```
